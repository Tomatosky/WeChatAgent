# Epic 13: 与作者共读

## 需求概述
在 DouDouChat（唯信）应用中引入“与作者共读”功能，支持用户导入多种格式的书籍，在阅读的同时能够与代表作者的 AI 好友进行上下文感知的实时交互，从而打造具有陪伴感和互动性的沉浸式阅读体验。

## 详细需求
0. 在主界面的左侧新增“与作者共读”的按钮（好友库按钮下面）。点击按钮之后，跳到图书管理页面。
1. 图书导入，支持 EPUB、PDF、MOBI、AZW/AZW3 和 TXT 格式文件。
2. 图书管理，支持对导入的图书进行增删改查，并且支持为特定图书绑定对应的作者（即现有的 AI 好友）。
3. 基础阅读器功能，支持打开图书并翻页浏览。
4. 目录导航功能，支持阅读中唤起并查看目录，通过目录快速跳转。
5. 进度记忆功能，支持在退出书籍时自动记录当前的阅读位置。
6. 伴读聊天窗口功能，在阅读窗口右侧提供聊天侧边栏，支持自动以当前页的内容作为背景上下文，和作者（AI好友）交流。
7. 选中文本交互功能，支持基于光标选中阅读区的一段文字后，将其作为直接的引用上下文和作者（AI好友）进行深度探讨。

## 统一约束与补充说明
1. **导入状态与阅读状态分离**：图书成功导入仅代表文件已被系统接收和保存，不代表一定可立即阅读、可提取文本、可进行目录导航或可进行引用交互。系统需区分至少以下状态：
   - `imported`：文件已保存，基础元数据已入库。
   - `processing`：正在解析正文、目录、封面或建立阅读索引。
   - `ready`：可正常打开阅读，且具备当前格式所支持的能力。
   - `limited`：可导入，但仅支持部分能力，例如只能分页浏览、无法提取正文、无法目录跳转或无法引用。
   - `failed`：解析失败或文件不可用，需向用户展示明确原因。
2. **格式能力按文件实际情况决定**：支持导入不等于支持全部高级能力。需要在 PRD 中明确下列降级规则：
   - EPUB/TXT：通常支持正文提取、目录或锚点定位、上下文聊天与引用交互。
   - PDF：仅在存在可提取文本层时支持正文引用与上下文聊天；扫描版或纯图片 PDF 允许阅读，但不保证支持引用与精准文本提取。
   - MOBI/AZW/AZW3：仅承诺对无 DRM、可解析的文件提供支持；若文件受 DRM 保护、结构异常或兼容性不足，应允许导入失败或进入 `limited/failed` 状态，并给出提示。
3. **不支持场景必须可解释**：以下情况必须有明确提示和降级处理，而不是 silent failure：
   - DRM/受版权保护的 Kindle 文件。
   - 密码保护或损坏的 PDF。
   - 无文本层的扫描件。
   - 目录缺失、封面缺失、编码异常、超大文件导致解析超时。
4. **聊天上下文必须可追溯**：伴读聊天和引用交互依赖“当前阅读位置对应的正文片段”。因此实现上必须建立可定位的数据结构，例如：
   - EPUB：章节锚点、CFI、段落区间。
   - PDF：页码、文本块区间、页内偏移。
   - TXT：章节索引、字符偏移或段落偏移。
5. **书籍维度会话隔离**：同一个 AI 好友绑定多本书时，每本书必须拥有独立的伴读会话上下文，不能直接复用普通单聊或其他书籍的聊天历史，否则会造成上下文串台。
6. **进度记录按格式定义**：不能统一抽象为“第几页”，应按格式分别记录页码、章节锚点、滚动偏移、字符偏移等信息，并在 UI 中以用户可理解的方式展示。
7. **前端导航方案需先决策**：当前项目主界面以单页状态切换为主。Epic 落地前需先明确“与作者共读”是新增真正路由页，还是沿用现有 `activeTab` / 主视图切换模式，避免设计与现有架构冲突。
8. **记忆系统污染控制**：阅读陪聊产生的上下文内容默认属于书籍会话语境，不应未经区分地写入用户长期画像。若后续接入长期记忆，需要为“书中讨论”与“用户真实偏好/事实”建立区分策略。

## 技术风险与注意事项
1. **格式解析兼容性**：在前端针对多种格式尤其是 PDF/EPUB 和 AZW 等私有或复杂格式的渲染方案（可结合 `pdf.js` 或 `epub.js` 等优秀开源库）。
2. **上下文容量管理**：传递当前页面上下文给大模型时，需控制 Token 消耗，确保不会造成单次聊天的成本和延迟过高。
3. **UI 适配**：确保阅读区与侧边栏的比例灵活可调，并且在大中小屏幕下维持“微信级”的交互美感。
4. **会话隔离与数据清理**：图书删除时需考虑正文缓存、目录索引、进度记录、伴读会话、封面文件等衍生资源的级联清理或保留策略。

## 数据库表结构

> 详细设计文档参见：`dev-docs/coding/epic_13/db_13_001_与作者共读表结构设计.md`

### 新增表：`books`（图书表）

| 字段名 | 类型 | 约束 | 描述 |
| ------ | ---- | ---- | ---- |
| `id` | Integer | PK | 主键 |
| `title` | String(255) | NOT NULL | 书名 |
| `author` | String(128) | 可空 | 书籍原作者 |
| `cover_url` | String(1024) | 可空 | 封面图路径 |
| `file_name` | String(255) | NOT NULL | 原始文件名 |
| `file_path` | String(1024) | NOT NULL | 文件存储路径（含后缀名表明格式） |
| `status` | String(32) | Default='imported' | 状态：`imported`/`processing`/`ready`/`limited`/`failed` |
| `status_detail` | Text | 可空 | 状态详情（如失败原因） |
| `ai_friend_id` | Integer | FK → friends.id，可空 | 绑定的伴读 AI 好友 |
| `reading_location` | String(255) | 可空 | 阅读进度定位值（页码/CFI 等），帮助用户下次恢复位置 |
| `create_time` | UTCDateTime | NOT NULL | 创建时间 |
| `update_time` | UTCDateTime | NOT NULL | 最后更新时间 |
| `deleted` | Boolean | Default=False | 软删除标记 |

### 扩展表：`chat_sessions`（新增字段）

| 字段名 | 类型 | 约束 | 描述 |
| ------ | ---- | ---- | ---- |
| `session_type` | String(32) | Default='normal' | 会话类型：`normal`（普通单聊）、`book_reading`（伴读聊天）等 |
| `knowledge_id` | Integer | 可空，无外键 | 通用知识 ID，根据 `session_type` 指向不同的表（如 `books.id`），不设外键以保持扩展性 |
