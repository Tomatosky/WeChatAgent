# Story 13-07: 基于选中内容的细粒度互动

> Epic: 与作者共读
> Priority: Medium

## 概述
提高页面上下文交流针对性精确度，允许用户直接选取屏幕上一句话、一段落进行标注并针对其发言提问。

## 用户故事

### US-1: 引用文本聊天
**作为** 用户，
**我希望** 当我选定某重点几行文本后，会有菜单让我可以立刻携带该段文本去提问旁边的人，
**以便** 获得它最具有针对性解释或评价。

## 功能范围

### 包含
- 提供对正在展现书页中文本的框选侦听操作 (`Selection` / `Range`)。
- 当高亮区域闭合并生成悬浮快捷弹层时展示对应的“作为引用去问作者”或类似的专属快速互动操作按钮。
- 将被框选的内容组装生成带有明确标记的特定 `Quote` 以直接打在下方聊天框进行组装或者发送。
- 在聊天发送结构中区分“页面背景上下文”和“用户显式引用片段”，避免两者在 prompt 里混淆。

### 不包含（后续迭代）
- 实现带有调色板、书眉批注并存档供以后使用的画线段高亮本地存储功能。

## 补充规则
- 本 Story 仅对“存在可选中文本层”的阅读内容承诺可用；对扫描版 PDF、纯图片页或当前渲染层无法稳定提供 Selection 的情况，只提供不可用提示，不强行实现。
- 移动端的手势交互需要服从所选渲染引擎的能力边界；若当前格式在移动端无法稳定弹出文本选择菜单，应明确降级。
- 选中内容过长时，系统需限制最大引用长度，并提示用户已截断。

## 验收标准
- [ ] AC-1: 鼠标（或手指按压）框选大段位于被看书籍上面的普通文字后能自动出现一个靠近附近位置的浮动菜单。
- [ ] AC-2: 该弹层中至少应当设有一项名为指向作者名字进行针对性探讨提问的首选项动作交互功能控制柄。
- [ ] AC-3: 被点击引发之后，不仅选中状态要取消，而且之前的内容已经呈现和附在侧边陪伴窗口输入框区域上方作为“背景块/引用块”，与要写字的问题构成同一个发送指令，当回复反馈送达用户界面处必须符合这段明确被高亮的文段为依托。
- [ ] AC-4: 对于不支持文本选择的内容，系统会明确提示“当前页面内容不支持引用提问”或等效文案，不出现无响应状态。
- [ ] AC-5: 当引用文本超过产品设定的长度上限时，系统会在发送前进行截断或阻止发送，并给出清晰提示；发送给后端的结构中能区分引用片段与普通输入文本。

## 附录：实现约束

### 1. 交互依赖前提
- 本 Story 仅在存在真实文本层且阅读引擎能稳定提供 `Selection / Range` 的情况下实现。
- EPUB / TXT 优先支持。
- PDF 仅在 text layer 可选中时支持；扫描版、图片型 PDF 或无法稳定选中的渲染页应明确降级。

### 2. 选区来源要求
- 选区监听必须发生在阅读器实际渲染文档内部，而不是只监听最外层 Vue 容器。
- 后续实现应通过阅读器适配层统一暴露选区读取能力，避免业务层直接耦合 `foliate-js` 的内部 DOM 结构。
- 适配层建议提供：
  - `getSelectionQuote(): QuoteContextResult | null`

### 3. 建议的引用结构
- 建议至少包含：
  - `text`
  - `locator`
  - `prefix`
  - `suffix`
  - `truncated`
  - `sourceType`
- `prefix / suffix` 用于给模型提供少量邻接语境，但不应替代整页背景上下文。

### 4. 与页面背景上下文分离
- 1307 的引用片段必须与 1306 的页面背景上下文分开传递。
- 请求结构中建议明确区分：
  - `user_message`
  - `page_context`
  - `quote_context`
- 不应把引用文本直接拼进普通输入框字符串后再让后端二次拆解。

### 5. 长度限制
- 引用片段应设置严格上限，避免用户一次框选过长内容。
- 建议限制为较短文本块，并在超长时执行截断或阻止发送；无论采用哪种策略，都必须给出明确提示。
- 若发生截断，结构中必须保留 `truncated=true` 标记。

### 6. UI 降级要求
- 无可选中文本时，不显示无意义的悬浮动作菜单。
- 若用户触发了引用入口但当前页不支持，应给出明确提示，例如“当前页面内容不支持引用提问”。
- 移动端若当前格式无法稳定触发系统选区或悬浮菜单，应允许仅桌面端支持，不能伪造一个不稳定交互。
