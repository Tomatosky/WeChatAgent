# Story 13-05: 目录导航与自动进度记录

> Epic: 与作者共读
> Priority: Medium

## 概述
增强阅读器的交互体验，提供文档目录索引机制以及在用户退出时自动进行书签点记录和恢复的能力。

## 用户故事

### US-1: 目录导航
**作为** 用户，
**我希望** 可以在阅读页悬浮呼出或侧拉面板打开电子书完整的章节目录结构并进行点击跳转，
**以便** 我寻找和定位想读的重点篇章。

### US-2: 自动记录位置
**作为** 用户，
**我希望** 每次中断或离开所阅读书籍时，系统都会默默保存我的具体滚动位置，并且在下次进入时直接恢复，
**以便** 我能始终无缝接续上次的阅读点。

## 功能范围

### 包含
- 支持格式的目录索引（TOC）解析与侧边呈现。
- 目录项具有交互可点击性，并且触发文档视图跳转。
- 定时或在脱离焦点时触发更新记录本地存储/数据库内基于该文件的阅读百分比位置进度标识。
- 针对不同格式定义不同的进度持久化模型，例如 EPUB 使用章节锚点/CFI，PDF 使用页码加页内偏移，TXT 使用字符或章节偏移。

### 不包含（后续迭代）
- 用户手工高亮保存的书签管理器。

## 补充规则
- 目录能力仅对具备可解析目录结构的书籍提供；TXT 或部分 PDF 若无 TOC，应展示空状态而不是报错。
- “恢复到上次位置”应以格式原生可稳定恢复的位置模型为准，不强制统一成“第几页”。
- 进度写入触发条件需覆盖主动返回、关闭窗口、切书、应用退出和异常中断后的最近一次已保存位置。

## 验收标准
- [ ] AC-1: 阅读界面能够提供唤起目录功能面板的操作（如图标或抽屉）。
- [ ] AC-2: 点击列表内的目录条目后，阅读视图准确渲染跳跃该卷或该章节的页数。
- [ ] AC-3: 关闭阅读界面，再重新点击进入同一本书时会自动跳转至退出时保留的那一页。
- [ ] AC-4: 对于不支持目录的图书，目录面板会展示“该文件无可用目录”之类的空状态提示，而不是空白或异常。
- [ ] AC-5: 对于不同格式，恢复逻辑以该格式支持的稳定定位方式生效；产品与测试文档不得统一用“页码”描述所有格式的进度恢复。

## 附录：实现约束

### 1. 统一定位模型
- 本 Story 的 `reading_location` 不应直接存储“第几页”这类面向 UI 的文案，而应存储可稳定恢复的紧凑定位值。
- 由于数据库字段为 `String(255)`，定位值必须保持简短，不得塞入长文本、目录路径全文或调试信息。
- 建议统一序列化为紧凑 JSON 字符串，并保留版本号，便于后续升级，例如：`{"v":1,"f":"epub","cfi":"..."}`。

### 2. 各格式稳定定位方式
- EPUB：优先使用 `CFI`，可附带 `fraction` 作为兜底显示进度。
- PDF：优先使用 `pageIndex`，可附带 `pageLabel`、`pageProgress` 或全书 `fraction` 作为恢复兜底。
- TXT：优先使用 `sectionIndex`，可附带 `charOffset` 或 `fraction`。
- 产品与接口层不得假设所有格式都能稳定映射成“页码”。

### 3. 阅读器适配层最小接口
- 后续实现应统一经由阅读器适配层提供能力，而不是在业务组件里直接读取 `foliate-js` 内部对象。
- 1305 至少需要以下接口：
  - `getToc(): TocItem[]`
  - `goToTocItem(target): Promise<void>`
  - `getCurrentLocation(): ReaderLocation`
  - `restoreLocation(location): Promise<void>`
- 对于无目录图书，`getToc()` 必须返回空数组，而不是抛错。

### 4. 建议的目录数据结构
- 建议统一为：
  - `label`
  - `href` 或内部跳转目标
  - `children`
  - `disabled`（可选，用于不可跳转项）
- 前端目录面板只消费归一化后的结构，不直接依赖不同格式原始 TOC 形态。

### 5. 进度写入触发时机
- 至少覆盖以下场景：
  - 主动点击返回
  - 切换到其他书
  - 浏览器/窗口关闭
  - 页面失焦或 `pagehide`
  - 阅读中位置变化后的节流写入
- 建议采用“位置变化后 debounce 写入 + 离开前强制写入”的组合，而不是只在点击返回时保存。

### 6. 接口建议
- 不建议复用图书基础信息更新接口来保存进度，建议单独提供阅读进度接口，例如：
  - `PATCH /api/books/{book_id}/reading-location`
- 请求体建议仅包含：
  - `reading_location`
  - `progress`（可选，便于列表展示）
  - `display_label`（可选，便于记录“第 12 页 / 第 3 章”这类摘要）
